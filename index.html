Dynamic Public Server Access & Management

Problem: Faced with restrictive ISP policies that blocked local server hosting, I needed a reliable and constantly accessible public link for my local Flask web application. Existing dynamic DNS solutions proved insufficient due to frequent IP changes and complex configurations.

Solution & My Role: As the sole developer, I engineered a robust system for establishing and maintaining public access to my local development server. This involved configuring Ngrok for secure tunneling, setting up Nginx as a reverse proxy, and using Gunicorn as a WSGI server to efficiently serve the Flask application. A crucial component was a Python script that dynamically fetched the current Ngrok public URL and automatically updated a GitHub Gist, which then informed my website's monitoring system. I also explored and implemented dynamic DNS solutions like `ddclient` with `no-ip.com` to achieve consistent domain-to-IP mapping, demonstrating adaptability when faced with ISP restrictions.

Key Technologies: Python, Flask, Gunicorn, Nginx, Ngrok, GitHub Gists, `requests` library, `ddclient`, `no-ip` API, Linux administration, Comprehensive Networking concepts (HTTP, TCP/IP, ports, tunneling, dynamic DNS).

Challenges & Learning: The biggest challenge was navigating and overcoming persistent ISP blocking measures and managing dynamic IP addresses. This required deep dives into network protocols, server configuration (Nginx, Gunicorn), and dynamic DNS services. Troubleshooting Ngrok tunnel stability and ensuring seamless integration with existing services further refined my Linux system administration and networking expertise.

Impact: This project successfully enabled continuous, reliable public access to my local web applications, overcoming significant connectivity hurdles. It facilitated consistent remote development and provided a stable platform for showcasing my Flask application, ensuring that my work was always accessible.

Impact: This project provides real-time uptime monitoring for my critical web services, including my personal Ngrok-exposed development environment. Polybar Multi-Monitor & Robust Launch Script

Problem & Goal: To ensure a consistent and reliable display of my Polybar status bar across various monitor configurations, particularly in multi-monitor setups common in an i3 window manager environment. The goal was to automate the correct launching of Polybar instances for each connected screen, preventing display issues.

Solution & My Role: As the sole developer, I crafted a concise yet powerful Bash script designed to intelligently launch Polybar. The script dynamically detects connected monitors and initiates a separate Polybar instance for each, ensuring optimal display and eliminating manual configuration headaches. It also includes a fallback mechanism for single-monitor or less conventional display environments.

Core Functionality:

  • Dynamic Monitor Detection: Utilizes `xrandr` to query and identify all currently `connected` display outputs.
  • Per-Monitor Launching: Iterates through each detected monitor, launching a dedicated Polybar instance and assigning it to the correct screen via the `MONITOR` environment variable.
  • Graceful Fallback: Includes an `else` condition to ensure Polybar launches correctly even if `xrandr` is not available or if the setup is a single display.
  • Reload Command: Uses `--reload` to ensure Polybar starts fresh, reflecting the latest configuration.

Key Technologies: Bash Scripting, `xrandr` (for display management), Polybar, Linux Command Line Interface (CLI), Environment Variables.

Challenges & Learning: The primary challenge involved reliably adapting Polybar's launch behavior to dynamic display changes in an i3 tiling window manager setup. This project solidified my understanding of shell scripting, `xrandr` output parsing, and managing processes with environment variables in a Linux context, ensuring a seamless user interface experience.

Impact: This simple yet effective script guarantees that my Polybar always launches and displays correctly, regardless of whether I'm using my laptop screen, an external monitor, or multiple external displays. It enhances the reliability and aesthetics of my Linux desktop environment by ensuring a consistent visual status bar.
